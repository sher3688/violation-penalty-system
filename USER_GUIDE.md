# 社區違規開罰管理系統使用指引

本指引供社區管理員、管理委員會承辦人員及住戶使用。系統的主要用途是將違規案件、現場照片、通知單、繳款、申訴與統計資料集中管理；通知單以 **A4 直向一頁、上方罰單聯、下方留存聯** 的格式產生，並在兩聯均顯示「住戶規約」作為法源依據。

> 本系統的資料權限以帳號角色與戶號為核心。管理員可處理所有案件；住戶帳號僅能讀取其綁定戶號的案件、照片與申訴資料。

## 1. 使用角色與功能範圍

| 角色 | 可執行的主要工作 | 不能執行的工作 |
|---|---|---|
| 管理員 | 建立住戶帳號、維護違規項目範本、登記案件、上傳照片、列印通知單、標記繳款、審核申訴、查看統計報表 | 無法以住戶身分提出申訴 |
| 住戶 | 查詢本人戶號案件與受保護照片、送出申訴、修改自己的登入密碼 | 無法查看其他戶號案件、建立案件、列印管理通知單、管理繳款或範本 |

## 2. 首次啟用與管理員設定

系統第一次開啟時，請由管理委員會指定的承辦人進入登入頁，依畫面的「首次設定」流程建立第一位管理員帳號。請使用可長期保管的帳號名稱與高強度密碼；完成後，往後的登入將使用該帳密，而非外部平台帳號。

管理員登入後，應先前往「住戶帳號」建立住戶資料。每個住戶帳號必須綁定一個戶號，建議同時登錄住戶姓名與聯絡電子郵件，以利後續辨識。若住戶搬離或帳號暫停使用，可在同一頁面停用帳號；停用不會刪除既有案件與繳款紀錄。

| 建議首次設定順序 | 操作位置 | 目的 |
|---|---|---|
| 1 | 首次設定／登入 | 建立第一位管理員帳號 |
| 2 | 住戶帳號 | 建立戶號及住戶登入帳號 |
| 3 | 違規項目範本 | 建立常用開罰類型與預設內容 |
| 4 | 個人帳號 | 確認管理員密碼與帳號資料 |
| 5 | 案件登記 | 以第一筆真實案件驗證通知單與列印設備 |

## 3. 建立與重複使用違規項目範本

「違規項目範本」可將經常使用的開罰內容保存為選單項目，避免每次案件登記都重複輸入。管理員可新增、編輯、啟用或停用範本；停用後不會再出現在新案件的選單中，但既有案件資料仍會完整保留。

每個範本宜包含清楚的違規名稱、預設說明、法源依據與建議罰款。例如，可建立「公共空間堆放雜物」範本，將法源依據填為「住戶規約」，並設定社區核定的建議金額。建立案件時，選取範本即可自動帶入內容；承辦人仍可依現場情況修改該筆案件的地點、說明、金額或法源文字，不會反向改動原始範本。

## 4. 登記違規案件與上傳照片

管理員可由側邊導覽進入「新增案件」。請先選擇戶號與違規項目範本，再確認違規發生日期時間、違規地點、違規說明及罰款金額。系統會為案件建立可追蹤的通知單號，初始狀態通常為「待繳款」。

現場照片會隨案件保存，並受權限保護。管理員可一次上傳最多 **6 張**照片，單張檔案不得超過 **8 MB**；支援 JPEG、PNG 與 WebP 格式。住戶登入後僅能查看與自己戶號相符案件的照片，其他帳號不能直接以檔案網址讀取。

| 欄位 | 填寫原則 |
|---|---|
| 戶號 | 從住戶清單選擇；系統以自然排序呈現戶號，例如 A-2 會排在 A-10 前面 |
| 違規類型 | 優先從已啟用範本選取，必要時可逐案調整 |
| 違規時間與地點 | 記錄能供住戶辨識與管理員追溯的資訊 |
| 違規說明 | 描述現場事實與改善或繳款通知，不宜使用模糊的責備性文字 |
| 罰款金額 | 應與住戶規約及社區決議一致；系統會拒絕負數或超出允許範圍的金額 |
| 法源依據 | 預設可使用「住戶規約」，也可依社區正式規範調整 |

## 5. 產生與列印 A4 通知單

案件建立後，請從案件清單或案件明細開啟「列印通知單」。系統會以固定 A4 直向頁面顯示上方的「罰單聯」與下方的「留存聯」，中間設有裁切虛線。兩聯均會列出管理單位、通知單號、戶號、違規內容、罰款、法源依據、現場照片區與簽收／簽章欄。

列印前請在瀏覽器列印對話框中選擇 **A4、直向、邊界設為無**。系統的列印樣式會自動隱藏返回與列印控制按鈕，避免其出現在紙本上。若社區採電子留存，建議同時輸出 PDF，並由管理單位依資料保存規範歸檔。

> 系統已以瀏覽器 PDF 列印引擎驗證：雙聯通知單會完整輸出於一張 A4 頁面。實際上線後，仍建議用第一筆案件在社區目標印表機或 PDF 印表機進行一次紙張、縮放與裁切線的最終確認。

## 6. 案件管理、繳款與結案

「案件列表」提供戶號、關鍵字、日期與狀態篩選，可用於快速查找承辦中的案件。案件狀態包括「待繳款」、「已繳款」、「申訴中」與「結案」。管理員應在案件明細中記錄實際繳款日期、金額與必要備註，再標記為已繳款；如案件已完成處理且無待審申訴，可再結案。

為維持紀錄一致性，申訴審核期間不應直接結案。若住戶提出申訴，案件將顯示為「申訴中」，管理員應先完成審核、輸入處理結果，再依結果更新案件狀態。所有管理與審核操作都應依社區正式規約及已授權的處理流程執行。

## 7. 住戶申訴流程

住戶以自己的帳密登入後，可在「我的案件」查看綁定戶號的案件。進入案件明細後，可輸入申訴理由並送出。送出後，管理員會在案件明細看見待處理的申訴內容與提交時間。

管理員完成檢視後，應在同一案件頁記錄申訴處理結果，例如維持原處分、撤銷、調整金額或要求補正，再將案件狀態調整為合適的後續狀態。住戶不會看到其他戶號的案件、照片、繳款或申訴資料。

## 8. 查看統計報表

管理員可開啟「統計報表」查看三類營運資訊：違規件數趨勢、各戶違規次數與繳款率。這些圖表會以系統中已建立的案件與繳款紀錄計算，並可協助管理委員會辨識高頻違規類型、需加強宣導的區域或待追蹤的繳款案件。

統計資料是行政管理輔助資訊，應搭配個案明細、住戶規約與社區決議解讀。若用於會議紀錄或對外公告，建議先移除可識別個人的戶號與照片資訊，並遵守社區的個資與資料保護規範。

## 9. 密碼、備份與日常維運

所有使用者皆可在「個人帳號」修改自己的密碼。管理員若需新增或停用住戶帳號，應使用「住戶帳號」頁面，而非共用管理員帳號。照片檔存放在部署環境指定的上傳目錄，案件、帳號、繳款與申訴資料則存放於 MySQL 資料庫；兩者都應納入定期備份計畫。

| 項目 | 建議頻率 | 管理原則 |
|---|---|---|
| MySQL 資料庫備份 | 每日或依社區案件量調整 | 保留可回復的資料庫備份，並定期測試還原程序 |
| 照片上傳目錄備份 | 每日或與資料庫備份同步 | 與資料庫備份一併保存，避免案件資料與照片失去對應 |
| 系統更新與版本快照 | 每次重大修改前後 | 保留可回復版本與變更紀錄 |
| 帳號權限檢查 | 每季或人員異動時 | 停用離任管理員與不再使用的住戶帳號 |

部署環境與備份位置的具體設定請參考專案內的 [部署文件](./DEPLOYMENT.md)。

## 11. 上線前快速檢核

在交由社區日常使用前，管理員應完成下列確認：已建立至少一位管理員與一個測試住戶帳號；已建立至少一個違規項目範本；已以實際照片測試上傳；已建立一筆案件並測試通知單 PDF／印表機輸出；已測試住戶提出申訴與管理員審核；已確認 MySQL 及照片目錄都納入備份。完成後，請刪除僅用於演練的測試帳號或測試案件，避免混入正式統計資料。
